# Identifiability Protocol

This protocol records competitor explanations for apparent subsystem emergence.

## Automated Checks Available Now

- coordinate choice: orthogonal-change sensitivity on selected linear benchmarks
- coarse-graining bias: coarse versus fine transport or Markov comparisons
- numerical truncation: refinement-span diagnostics
- transient coincidence: early-versus-late leakage mismatch score

## Manual Or Deferred Checks

- clustering artifact: inspect whether discovered subsystems merely mirror a clustering heuristic
- post hoc projection: freeze the projection before held-out evaluation
- application-facing coarse graining: compare mobility and clickstream claims against coarse grouped windows or reduced named-state slices before treating them as robust

## Acceptance Rule

A positive subsystem claim should survive at least one automated artifact check and should not fail the available manual protocol without explanation.
